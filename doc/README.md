# doc/

Assets and scripts used to generate the fdb demo GIF and other documentation visuals.

## Files

| File | Purpose |
|---|---|
| `demo.tape` | [VHS](https://github.com/charmbracelet/vhs) tape script — defines what to type, timing, and visual settings for the demo recording |
| `fdb-mock.sh` | Fake `fdb` that returns pre-canned realistic output — makes the recording deterministic (no real device needed) |
| `fdb-demo.sh` | Real `fdb` wrapper — strips noisy Dart/pub log lines; use this instead of the mock for a live recording |
| `fdb-demo-shell.sh` | Shell launcher used by VHS for a live demo session (sets up the alias and working directory) |
| `fdb-demo-env.sh` | Environment setup sourced for a live demo (same as above, without exec) |
| `zdotdir/.zshrc` | Minimal `.zshrc` loaded by VHS via `ZDOTDIR` for a clean shell environment |
| `demo.gif` | Output — the recorded demo animation (committed, regenerated by the tape) |
| `fdb-banner.png` | Static banner image used in the README |
| `fdb-banner.html` | Source HTML for generating the banner |
| `header.svg` | SVG header graphic |
| `how-it-works-light.svg` | Diagram for the README (light mode) |
| `how-it-works-dark.svg` | Diagram for the README (dark mode) |
| `README.agents.md` | Guide for using fdb with AI coding agents |

## Re-recording the demo GIF

### Prerequisites

- [VHS](https://github.com/charmbracelet/vhs) installed (`brew install vhs`)
- No real device needed — the tape uses `fdb-mock.sh`

### Run

```bash
vhs doc/demo.tape
```

This writes the output to `doc/demo.gif`.

### Live recording (with a real device)

If you want to record against an actual running app instead of the mock, edit `demo.tape` and change the alias line to point at `fdb-demo.sh`:

```
Type "alias fdb='$(pwd)/fdb-demo.sh'"
```

Then make sure `fdb` and `flutter` are on your `PATH` before running `vhs`.
